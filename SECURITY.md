# Security Policy

## Reporting a Vulnerability

If you discover a security issue in Allors TRAM, please report it privately through GitHub's [Security Advisories](https://github.com/allors/tram-dotnet/security/advisories/new). Do not open a public issue.

If you cannot use GitHub Security Advisories, email **legal@allors.com** with the details.

Please include:

- A description of the issue and its impact.
- Steps to reproduce (proof-of-concept code, configuration, or test case).
- Affected version(s).
- Any suggested mitigation.

## Response Targets

| Stage | Target |
| --- | --- |
| Acknowledgment of the report | within 72 hours |
| Initial assessment | within 7 days |
| Fix or mitigation released | within 30 days for high-severity issues |

These are targets, not guarantees. We will keep you informed of progress.

## Supported Versions

While Allors TRAM is in `0.x`, only the latest minor version receives security fixes. Once `1.0` ships, this table will be updated.

| Version | Supported |
| --- | --- |
| 0.1.x   | Yes       |
| < 0.1   | No        |

## Disclosure

We coordinate disclosure with reporters. Once a fix is available, we publish a security advisory on GitHub with credit to the reporter (unless they prefer to remain anonymous).
